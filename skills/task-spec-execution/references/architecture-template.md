# Architecture Template

Use for stable behavior boundaries and long-lived design documents.

## Recommended Structure

```md
# <Topic>

## Background
- Why this design is needed

## Goals
- What this design must achieve

## Non-goals
- What is explicitly out of scope

## Design
- Core structure, boundaries, and behavior rules

## Risks / Edge Cases
- Important constraints, failure modes, or special situations

## Related Documents
- Links to task, spec, or context docs
```

## Notes

- Do not put temporary execution notes here.
- Do not track task status here.
- If the content is still exploratory or unvalidated, prefer `docs/context/`.
